Hypercolumns for Object Segmentation and Fine-grained Localization (CVPR 2015 Oral)

#####Abstract
CNN最后一层提取的feature太过于sparse，所以不适合精确位置的定位。之前那些层次可能在定位上会比较精确，但是不一定在提取一些语意信息上面会非常好
如何理解CNN提取出来的feature？

#####Introduction
reasoning at multiple levels of abstraction and scale is necessary.
Author认为一个CNN每一层的卷积或者pooling之后的feature map也就像是一个Image pyramids
立足于两个很具体的问题：

  1. simultaneously detection and recognition（maybe for word detection and recognition）
  2. detecting an object and localizing its parts (keypoint or segment out each part)
    #####Related Work
  3. CNN: combine features across multiple levels
  4. detection and segmentation
  5. pose estimation and part labeling
    #####Methodology Part:
    ######问题的设定：
    一个object detection system已经得到很多检测的结果，每一个检测的结果有：bounding box, 所属类别的标签，score。这些检测的结果是已经进行非极大值抑制之后的结果（大部分的detection都会有NMS这一步）。我们现在所想要做的是在已知bounding box的前提下分割出物体，物体的每个部分或者关键点。
    Input: 略微扩大之后的bounding box
    Output: Heatmap(different tasks have different definitions of the heatmap), size: 50x50
    ######Hypercolumn representation
    keyword: bilinear interpolation, upsample feature map
    ######Grid Classifiers
    Feature maps of convolutions and poolings don’t encode any information of location.
    这一段也就是说，fully connected layer出来的feature是整个bounding box里面的共享的，是instance-specific。而heatmap上面每一个位置都可以训练一个location-specific的classifier，这样一来global的feature + location-specific classifiers就可以produce出一个instance-specific的prior。
    然而有three drawbacks that the author didn’t train 2500 classifiers. They adopted grid classifiers instead.
    How grid classifiers work
    50x50的feature maps分成K x K的grid，在训练的过程中就把所有的kth grid里面的像素拿出来训练第k个classifier。
    在预测的过程中，每一个位置的结果都是K x K个classifier的一个linear combination（双线性插值的一个变形）
    ######Efficient implementation and Representation as a neural network
    We don’t have to resize the feature map to 50 x 50. 之前说了TMD这么多的废话，最后还是把训练grid classifier的这样的一件事情转化成了训练一个卷积层，只不过这个卷积层的channel是K X K

######个人总结
这篇文章最主要的贡献在于：

  1. 提出了Hypercolumns feature，认为CNN fully connected layer输出的feature具有很好的classify label这样的一个特性，但是过于sparse，并不适合localization的这样一个过程
  2. 提出了一个由bounding box到segmentation的这样的一个CNN framework。
  3. 效果非常好
    我觉得值得考虑的几件事情：
  4. upsample是直接进行线性插值？
  5. K x K grid classifiers的训练好像并没有分开，而是全部结合在了一起。
  6. segmentation这样一件事情的起始点还是在有了比较好的bounding box的基础上进行
  7. 复现原文结果的代价比较大
